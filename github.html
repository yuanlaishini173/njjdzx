GitHub 到底是什么？

用三个比喻，让你秒懂 GitHub

📚

代码的"图书馆"

GitHub 就像一个巨大的代码图书馆，全世界的程序员都把自己的代码放在这里。 你可以借阅（下载）别人的代码，也可以把自己的代码存进去，还能和别人一起"写书"（协作开发）。

🔄

项目的"时光机"

GitHub 能记录代码的每一次修改，谁改了什么、什么时候改的，一清二楚。 改错了还能"穿越回去"，就像游戏存档一样，可以随时回退到之前的版本。

🤝

团队的"协作平台"

几十上百人一起开发同一个项目，怎么不乱套？ GitHub 提供了完整的协作工具，让大家能有序地分工合作、讨论问题、审核代码。

核心概念

这些名词，一次搞懂

刚接触 GitHub 会看到很多新名词，其实原理都很简单

🗄️ Repository (仓库 / Repo)

一个项目就是一个仓库。可以理解为项目文件夹，里面放着项目的所有文件和修改历史。 有公开的（任何人都能看）和私有的（只有你授权的人能看）。

📋 Commit (提交)

一次保存就是一次 commit。每次提交都会记录"谁在什么时间改了什么"， 还可以写一段说明，告诉别人这次改了啥。就像写日志一样。

🌿 Branch (分支)

想象你在画一幅画，想试试新的配色，但又怕改坏了。 分支就是"复制一份出来试试"，试好了再合并到主版本上，不影响原来的。

🔀 Pull Request (PR / 合并请求)

"我改好了，请求合并到主版本"。发 PR 就是告诉大家：我这边做完了， 请来 review（审核）一下，没问题就合并进去。

🍴 Fork (复刻)

看到别人的项目不错，想自己改着玩？Fork 就是"复制一份到我自己名下"， 你可以随便改，不会影响原来的项目。改得好还可以申请合并回去。

⭐ Star (星标)

就像朋友圈点赞。看到好的项目点个 Star，表示你喜欢、收藏了。 项目的 Star 数量越多，说明越受欢迎。

🐛 Issue (议题)

发现 Bug 了？有新想法？都可以提 Issue。 相当于项目的"留言板/工单系统"，大家在这里讨论问题、追踪进度。

⚡ GitHub Actions

自动化工具。可以设置"代码一提交，就自动测试、自动部署"， 省去很多重复劳动，相当于你的"自动化助手"。

典型工作流：一次完整的开发

1

创建/克隆仓库

新项目：在 GitHub 上创建一个新仓库
已有项目：把仓库克隆（下载）到自己电脑上

git clone https://github.com/xxx/xxx.git
2

创建新分支

不要直接在主分支上改！新建一个分支，在新分支上干活， 这样改坏了也不怕，主版本始终是稳定的。

git checkout -b my-feature
3

写代码，提交修改

改完一部分就提交一次，写清楚改了什么。 小步快跑，比一次改一大堆要好维护得多。

git add .
git commit -m "添加了登录功能"
4

推送到 GitHub

把本地的修改推送到 GitHub 上，这样别人就能看到你的工作了， 也相当于在云端备份了一份。

git push origin my-feature
5

发起 Pull Request

功能做完了，发起 PR 请求合并。写清楚做了什么、怎么测试的， 让队友来 Code Review（代码审查）。

6

Code Review & 合并

队友审核代码，有问题就讨论修改。没问题了就合并到主分支。 一次完整的开发就完成了！

常见问题

你可能想问的

Git 和 GitHub 是一回事吗？ +

不是！很多新手会搞混：

• Git：是一个"版本控制软件"，装在你电脑上，用来管理代码历史
• GitHub：是一个"网站/平台"，把大家的 Git 仓库放在云端，方便共享和协作

打个比方：Git 是你电脑里的 Word，GitHub 是百度网盘/腾讯文档。 你用 Word 写文档，存在网盘里，可以和别人共享协作。

GitHub 免费吗？要花钱吗？ +

个人使用基本免费！

GitHub 的免费版包括：
• 无限数量的公开仓库和私有仓库
• 最多 3 个协作者（私有仓库）
• GitHub Pages 免费静态网站托管
• 每月 2000 分钟的 Actions 自动化额度

个人开发者、学生、小团队，免费版完全够用。 如果需要更多协作人数和企业功能，才需要付费。

我不会写命令行，能用 GitHub 吗？ +

当然可以！有很多方式可以不用命令行：

• GitHub Desktop：官方出品的桌面客户端，全图形界面，点鼠标就行
• Trae / VS Code：编辑器里内置了 Git 功能，可视化操作
• GitHub 网页：直接在网页上上传文件、编辑、提交

命令行只是其中一种方式，而且不是必须的。新手推荐先用图形界面工具，等熟悉了再学命令行也不迟。

代码放 GitHub 上安全吗？别人会偷吗？ +

看你设为公开还是私有：

• 公开仓库 (Public)：任何人都能看、能下载。适合开源项目、学习分享
• 私有仓库 (Private)：只有你和你邀请的人才能看。适合公司项目、个人私活

GitHub 的安全性是有保障的，毕竟是微软旗下的产品。 只要你不把密码、密钥等敏感信息提交到公开仓库，就不用担心。

新手怎么开始学 GitHub？ +

推荐学习路径：

第 1 步：注册账号，逛一逛
先去 github.com 注册个账号，随便看看，搜搜感兴趣的项目，点几个 Star。

第 2 步：创建你的第一个仓库
在网页上点 New Repository，创建一个仓库，试着上传几个文件。

第 3 步：用桌面客户端
下载 GitHub Desktop，试着克隆仓库、修改、提交、推送。

第 4 步：学基础命令（可选）
等图形界面用熟了，想学得更深，再学 Git 命令行。

国内访问 GitHub 慢怎么办？ +

国内访问 GitHub 确实可能比较慢，几个解决方案：

• 使用镜像站：比如 hub.fastgit.xyz 等镜像加速下载
• Gitee 码云：国内版的 GitHub，速度快，功能类似
• GitCode：CSDN 旗下的代码托管平台
• Trae 内置：Trae IDE 里可以直接操作 GitHub，有优化

另外，GitHub 在国内也在不断优化访问速度，日常使用基本没问题。
